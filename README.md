
# Интерактивный веб-калькулятор на JavaScript

## 1. Описание проекта

Данный проект представляет собой интерактивный веб-калькулятор, разработанный с использованием HTML, CSS и современного JavaScript (ES6+). Приложение реализует базовые математические операции с валидацией ввода и обработкой ошибок. Проект демонстрирует принципы модульной архитектуры, разделения ответственности и безопасной работы с пользовательским вводом.

## 2. Функциональность

### Поддерживаемые операции:
- Арифметические операции: сложение (+), вычитание (-), умножение (*), деление (/)
- Работа с десятичными дробями: ввод чисел с плавающей точкой (.)
- Отрицательные числа: поддержка отрицательных значений
- Очистка: полная очистка текущего выражения (кнопка C)
- Вычисление результата: получение результата выражения (кнопка =)

### Ограничения:
- Запрещён ввод двух операторов подряд (кроме минуса в начале числа)
- Нельзя ввести несколько точек в одном числе
- Выражение не может начинаться с оператора (кроме минуса)
- Запрещён оператор в конце выражения

### Обработка ошибок:
- Некорректное выражение: при попытке вычислить недопустимое выражение отображается сообщение "Ошибка"
- Деление на ноль: обрабатывается через try-catch, отображается "Ошибка"
- Бесконечный результат: проверяется через `isFinite()`, при некорректном результате выбрасывается ошибка
- Состояние ошибки: после ошибки калькулятор блокируется до нажатия кнопки очистки (C)

### Класс Calculator

Класс `Calculator` инкапсулирует всю логику работы калькулятора:

**Поля класса:**
- `display` — ссылка на DOM-элемент поля ввода (дисплей)
- `expression` — строка текущего математического выражения
- `hasError` — флаг состояния ошибки

**Основные методы:**

| Метод | Назначение |
|-------|------------|
| `init()` | Инициализация: навешивает обработчики событий на кнопки |
| `handleButtonClick(value)` | Распределяет действия в зависимости от нажатой кнопки |
| `addToExpression(value)` | Добавляет символ в выражение с валидацией |
| `calculate()` | Вычисляет результат выражения |
| `safeEvaluate(expr)` | Безопасное вычисление с проверкой корректности |
| `isValidExpression(expr)` | Валидация математического выражения |
| `clear()` | Сброс состояния калькулятора |
| `updateDisplay()` | Обновление отображения на дисплее |
| `showError(message)` | Отображение сообщения об ошибке |
| `isOperator(char)` | Проверка: является ли символ оператором |
| `isLastCharOperator()` | Проверка последнего символа на оператор |
| `hasDuplicateDot()` | Проверка на дублирующую точку в числе |

## 4. Логика работы

### Последовательность действий при нажатии кнопок

1. **Пользователь нажимает кнопку** → Срабатывает событие `click`
2. **`init()`** → Обработчик вызывает `handleButtonClick(button.textContent)`
3. **`handleButtonClick(value)`** → Определяет тип кнопки:
   - Если была ошибка и нажата не "C" → вызывает `clear()`
   - Если "C" → вызывает `clear()`
   - Если "=" → вызывает `calculate()`
   - Иначе → вызывает `addToExpression(value)`

### Как работает валидация ввода

**Метод `addToExpression(value)` проверяет:**

1. **Два оператора подряд**:
   ```javascript
   if (this.isOperator(value) && this.isLastCharOperator()) {
       // Заменяет последний оператор на новый
       this.expression = this.expression.slice(0, -1) + value;
   }
   ```

2. **Дублирующая точка в числе**:
   ```javascript
   if (value === "." && this.hasDuplicateDot()) {
       return; // Игнорирует ввод
   }
   ```
   Метод `hasDuplicateDot()` разбивает выражение по операторам и проверяет последнюю часть на наличие точки.

3. **Выражение начинается с оператора** (кроме минуса):
   ```javascript
   if (this.isOperator(value) && !this.expression && value !== "-") {
       return; // Игнорирует ввод
   }
   ```

**Метод `isValidExpression(expr)` проверяет:**
- Отсутствие двух и более операторов подряд (регулярное выражение `/[+\-*/]{2,}/`)
- Отсутствие оператора в конце строки (регулярное выражение `/[+\-*/]$/`)

### Как происходит вычисление

1. **Пользователь нажимает "="** → Вызывается `calculate()`

2. **Проверка условий**:
   ```javascript
   if (!this.expression || this.hasError) return;
   ```

3. **Безопасное вычисление**:
   ```javascript
   try {
       const result = this.safeEvaluate(this.expression);
   ```

4. **Валидация выражения** (`safeEvaluate`):
   ```javascript
   if (!this.isValidExpression(expr)) {
       throw new Error("Некорректное выражение");
   }
   return eval(expr);
   ```

5. **Проверка результата**:
   ```javascript
   if (!isFinite(result)) {
       throw new Error("Некорректный результат");
   }
   ```

6. **Успешное вычисление**:
   - Результат сохраняется в `this.expression`
   - Флаг ошибки сбрасывается: `this.hasError = false`
   - Дисплей обновляется: `this.updateDisplay()`

7. **Обработка ошибки**:
   ```javascript
   catch (error) {
       this.showError("Ошибка");
   }
   ```
   - На дисплее отображается "Ошибка"
   - Устанавливается флаг `this.hasError = true`
   - Выражение очищается
